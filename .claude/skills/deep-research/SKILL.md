# Deep Research スキル

## タスク

$ARGUMENTS について徹底調査する:

### 1. 関連ファイルの網羅的発見
- Glob でディレクトリ構造を把握
- Grep でキーワード検索（複数の同義語・関連語）
- 候補ファイルを優先度付け（コア / 周辺 / テスト）

### 2. 主要ファイルの読み込み
- コア実装を読み、設計意図を理解
- 周辺ファイルで利用パターンを抽出
- テストから期待される挙動を逆引き

### 3. 依存関係の追跡
- import / require / 関数呼び出し関係
- 上流: このトピックが依存する基盤コンポーネント
- 下流: このトピックを利用する箇所

### 4. パターンと落とし穴の抽出
- プロジェクト固有の規約（命名 / エラーハンドリング / 抽象化レベル）
- エッジケース処理の有無
- TODO / FIXME / 技術的負債のマーカー

## 出力フォーマット

以下の構造で要約をメイン会話に返す（詳細な分析結果はサブエージェント側に留める）:

```markdown
## 全体像
1-2 段落で要約

## 主要コンポーネント
- `path/to/file.ext`: 役割と責務（1 行）
- ...

## パターン / 規約
- 命名: ...
- エラーハンドリング: ...
- ...

## 注意点
- バグの匂い / TODO / 技術的負債（ファイル:行番号付き）
```

## 起動例

- `/deep-research authentication system`
- `/deep-research how database transactions are handled`
- 「認証フローについて徹底的に調べて」
