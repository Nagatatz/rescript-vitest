# トークン最適化ガイドライン

## モデル選択基準

タスクの複雑さに応じて適切なモデルを選択し、コストとレイテンシを最適化する。

### Haiku（軽量タスク向け）

- ファイル検索・パターンマッチング
- ドキュメント・コメントの作成
- 単純なファイル編集（typo修正、import追加等）
- コードフォーマット確認
- 定型的なボイラープレート生成

### Sonnet（標準タスク向け・デフォルト）

- 複数ファイルにまたがる実装
- PRレビュー・コードレビュー
- バグ修正・デバッグ
- テストコード作成
- リファクタリング

### Opus（高度なタスク向け）

- アーキテクチャ設計・技術選定
- セキュリティ分析・脆弱性検出
- 複雑なデバッグ（複数システム間の問題）
- パフォーマンス最適化の設計
- 大規模リファクタリングの計画

## コンテキスト節約のプラクティス

1. **必要なファイルのみ読み込む**: 関連性の低いファイルを無駄に読まない
2. **サブエージェントの活用**: 独立したタスクは `.claude/agents/` のエージェントに委譲する
3. **段階的な探索**: まず Glob/Grep で対象を絞り、必要なファイルだけ Read する
4. **出力の簡潔化**: 冗長な説明を避け、要点を端的に伝える
