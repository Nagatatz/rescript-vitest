# add-rule スキル

ユーザーが「ルール化してください」「ルールに追加して」「これをルールにして」等と発言した際に、会話の文脈からルール内容を抽出し、`.claude/rules/` に追加するスキル。

## トリガー

以下のようなユーザー発言で起動する:
- 「ルール化してください」
- 「ルールに追加して」
- 「これをルールにして」
- 「rules に書いて」
- 「CLAUDE.md に追記して」（ルール関連の場合）

## 手順

### ステップ 1: ルール内容の特定

会話の直前の文脈から、ルール化すべき内容を特定する。以下を確認:

- **何をルール化するか**: 具体的なルール内容（コーディング規約、プロセス、命名規則等）
- **既存ルールとの関連**: `.claude/rules/` 配下の既存ファイルに追記すべきか、新規ファイルを作成すべきか

### ステップ 2: 配置先の決定

`.claude/rules/` 配下の既存ファイルを一覧し、適切な配置先を決定する:

1. `ls .claude/rules/` で既存ルールファイルを確認
2. 各ファイルの内容を読み、追記すべきファイルがあるか判断
3. 該当するファイルがなければ新規ファイルを作成

**新規ファイルの命名規則:** ケバブケースで内容を端的に表す名前にする（例: `naming-conventions.md`, `error-handling.md`）

### ステップ 3: ルール内容の作成

以下のフォーマットでルールを記述する:

```markdown
## <ルールの見出し>

**以下は強制的な行動指示であり、例外なく従うこと。**

<ルール本文>

**例外:** <例外がある場合に記載>
```

- 既存ファイルに追記する場合: ファイル末尾に適切なセクションとして追加
- 新規ファイルを作成する場合: `# <タイトル>` ヘッダーから始める

### ステップ 4: CLAUDE.md への反映

ルールが CLAUDE.md の内容にも影響する場合:
- CLAUDE.md の該当セクションにも反映する
- 新規ルールファイルを作成した場合、CLAUDE.md に `@.claude/rules/` 参照があればリストに追加する

### ステップ 5: 確認

- 追加・変更した内容をユーザーに提示し、確認を得る
- 承認後、ファイルを保存する

## 注意事項

- 既存ルールと矛盾する内容が指示された場合、ユーザーに確認してから対応する
- ルールは日本語で記述する（既存ルールファイルのスタイルに合わせる）
- 「強制的な行動指示」のフレーズは、本当にプロセスを強制する必要がある場合のみ使用する
- 軽い推奨事項の場合は「〜すること」程度の表現で十分
