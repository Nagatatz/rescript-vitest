# コードコメント規約

すべてのソースコードにドキュメントコメントを記述すること。

## 必須コメント

- **クラス/モジュールレベル:** すべてのクラス・モジュール・型定義にドキュメントコメントを付与する
  - クラスの責務と役割を 1〜3 文で記述する
  - 関連するクラス・モジュールへの参照を適宜追加する

- **関数/メソッドレベル:** 以下の条件に該当するものにドキュメントコメントを付与する
  - `public` / エクスポートされた関数で、名前だけでは目的が明確でないもの
  - パラメータが 2 つ以上ある関数（パラメータと戻り値の説明を記述）
  - 複雑なロジック（分岐が多い、外部プロセス呼び出し、正規表現処理など）

- **インラインコメント:** 以下の箇所にコメントを追加する
  - 非自明なアルゴリズムやワークアラウンド
  - 正規表現パターンの説明
  - マジックナンバーや定数の意味

## 省略可能なケース

- インターフェースの実装メソッドで、親の説明が十分な場合
- Getter / Setter のみのプロパティ
- データクラスのプロパティ（名前が自明な場合）
- テストクラス・テストメソッド（テスト名が説明的であれば不要）
