# Permission Mode 運用規約

Permission Mode（Plan Mode / auto / sandbox / accept-edits）と steering ワークフローの住み分け。

## 判断フロー

| 作業の種類 | 推奨モード |
|----------|----------|
| タイポ修正、1 行の設定変更 | 通常モード（直接実行） |
| 軽微な変更（差分を 1 文で説明できる） | Plan Mode |
| 中規模以上（複数ファイル、不慣れなコード） | steering ワークフロー |
| CI / 非対話バッチ | auto mode (`--permission-mode auto`) |
| 信頼できないコード実行 / 大量ファイル操作 | sandbox (`--sandbox`) |

**Plan Mode** はセッション内の軽量サイクル（計画→確認→実行）。**steering** は `.steering/` 永続化 + worktree 隔離の重量プロセス（`steering-workflow.md`）。Plan Mode で「中規模以上」と判明したら steering に昇格する。

## Think Before Coding（曖昧指示への対応）

ユーザー要求に複数の解釈余地がある場合、**黙って一つを選んで実装してはならない**。計画フェーズで:

- **仮定を明示する**（「X と解釈して進めます」と宣言してから着手）
- **トレードオフを表面化する**（複数アプローチが妥当なら利点・欠点を提示しユーザーに判断を仰ぐ）
- **不明点で立ち止まる**（「何が分からないか」を名指しで質問。曖昧なまま進めない）
- **より簡潔な代替があれば提案する**（`minimal-change.md` と連動）

**例外:** 解釈の幅が小さくユーザー意図が明白な場合（タイポ修正等）は宣言を省略してよい。
