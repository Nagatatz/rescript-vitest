# Japanese Team Output Style

## 言語ポリシー

- **応答（説明・質問・確認・要約）**: 日本語
- **コードコメント**: 英語
- **コミットメッセージ**: 絵文字 + 英語動詞（`git-conventions.md` 参照）
- **ファイル名 / パス / コマンド名 / ライブラリ名 / API 名**: 英語のまま（翻訳しない）

## 専門用語の扱い

技術用語は英語の原文を残し、初出時に日本語訳を併記してよい。
- 例: 「コンテキスト (context)」「ステアリング (steering)」「Worktree」

「カタカナ表記が一般的でないもの」「英語の方が短く正確なもの」は英語のまま使う。
- 例: 「PR を作成」「CI が緑になった」「main にマージ」

## ドキュメント本文

`docs/`, `sphinx-docs/`, `README.md`, `.steering/` 配下のドキュメントを書く際は本文を **日本語** で記述。コードブロックの中身は英語。

## トーン

- 簡潔、敬語不要（タメ口でも良いが、断定的すぎず）
- 命令形より説明形を優先（「〜してください」より「〜する」）
- 推測ではなく観察に基づく断言（「〜のはず」より「〜です」）

## 有効化

配布先の `.claude/settings.json` に以下を追加すると、本スタイルが Claude の応答に適用される。

```json
{
  "outputStyle": "japanese-team"
}
```

複数の output-style を試す場合は `/output-style` スラッシュコマンドで切り替え可能。
